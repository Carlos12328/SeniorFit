# SeniorFit MVP Testável

Este projeto é uma reconstrução do MVP "SeniorFit" (acompanhamento físico de idosos), com foco em **testabilidade** para avaliação acadêmica. A implementação foi feita do zero, seguindo as "Regras de Ouro" de simplicidade e didática, e as "Regras de Ouro" de arquitetura e stack.

## 1. Visão Geral

O SeniorFit é uma aplicação para gerenciar o acompanhamento físico de idosos. Ele permite o registro de usuários com diferentes perfis (Admin, Acompanhante, Profissional), o cadastro de idosos, o registro de atividades físicas e a consulta da frequência de atividades. As regras de acesso são rigorosas para garantir a privacidade e a segurança dos dados.

### Perfis de Acesso

*   **Admin**: Acesso total. Pode gerenciar usuários, idosos e todas as atividades.
*   **Acompanhante**: Pode visualizar **apenas** as informações do idoso vinculado a ele e registrar atividades para esses idosos.
*   **Profissional**: Pode registrar idosos, registrar atividades para qualquer idoso e visualizar o histórico de atividades de qualquer idoso.

## 2. Stack Tecnológica

*   **Frontend**: HTML, CSS e JavaScript puro.
*   **Backend**: Node.js com Express.
*   **Banco de Dados**: SQLite (usando o pacote `sqlite`).

## 4. Configuração e Execução

### Pré-requisitos

*   Node.js (versão 18 ou superior)
*   npm (gerenciador de pacotes do Node.js)
*   Google Chrome (para testes Robot Framework)

### Instalação

1.  **Clone o repositório (ou descompacte o ZIP):**

    ```bash
    git clone <URL_DO_REPOSITORIO>
    cd SeniorFit
    ```

    Ou, se você recebeu um ZIP, descompacte-o em uma pasta chamada `SeniorFit`.

2.  **Instale as dependências:**

    ```bash
    npm install
    ```

3.  **Configure as variáveis de ambiente:**

    Crie um arquivo `.env` na raiz do projeto (na mesma pasta do `package.json`) e adicione:

    ```
    JWT_SECRET=seniorfit_chave_academica
    PORT=3000
    ```

    Você pode copiar o `.env.example` para `.env`.

### Execução da Aplicação

Para iniciar o servidor backend:

```bash
npm start
```

O servidor estará disponível em `http://localhost:3000`. Você pode acessar a aplicação frontend abrindo `http://localhost:3000/login.html` no seu navegador.

## 5. Testes

O projeto inclui três tipos de testes, focados em diferentes aspectos da testabilidade:

*   **Testes Unitários (Jest)**: Para validações de regras de negócio e endpoints da API.
*   **Testes de API (Postman)**: Uma coleção para testar os endpoints da API manualmente ou via Newman.
*   **Testes de UI (Robot Framework)**: Para validar fluxos de usuário no frontend.

### Executando Testes Unitários

```bash
npm run test:unit
```

Ou para todos os testes (unitários e de API com Supertest):

```bash
npm test
```

### Executando Testes de API (Postman)

1.  **Importe a coleção**: Abra o Postman e importe o arquivo `tests/postman/SeniorFit.postman_collection.json`.
2.  **Variáveis de Ambiente**: A coleção usa variáveis de ambiente (`baseUrl`, `adminToken`, etc.). Certifique-se de que `baseUrl` esteja configurado para `http://localhost:3000`.
3.  **Execute os testes**: Você pode rodar a coleção inteira ou requisições individuais. Os scripts de `Pre-request` e `Tests` dentro das requisições geram e capturam tokens automaticamente.

### Executando Testes de UI (Robot Framework)

1.  **Instale o Robot Framework e SeleniumLibrary**: Se ainda não tiver, instale:

    ```bash
    pip install robotframework robotframework-seleniumlibrary
    ```

2.  **Baixe o WebDriver**: Certifique-se de ter o ChromeDriver compatível com sua versão do Chrome e que ele esteja no seu PATH ou na pasta do projeto.
3.  **Execute os testes**: Com o servidor (`npm start`) rodando em `http://localhost:3000`:

    ```bash
    robot tests/robot/login.robot
    robot tests/robot/fluxos_principais.robot
    ```

### `.vscode/launch.json`

No mesmo diretório `.vscode`, adicione o arquivo `launch.json` com o seguinte conteúdo:

```json
{
  "version": "0.2.0",
  "configurations": [
    {
      "type": "node",
      "request": "launch",
      "name": "Launch Backend",
      "program": "${workspaceFolder}/backend/server.js",
      "runtimeArgs": [
        "--enable-source-maps"
      ],
      "envFile": "${workspaceFolder}/.env",
      "preLaunchTask": "Start Backend"
    },
    {
      "type": "chrome",
      "request": "launch",
      "name": "Launch Frontend",
      "url": "http://localhost:3000/login.html",
      "webRoot": "${workspaceFolder}/frontend"
    },
    {
      "type": "node",
      "request": "launch",
      "name": "Debug Unit Tests",
      "program": "${workspaceFolder}/node_modules/jest/bin/jest.js",
      "args": [
        "--runInBand",
        "--verbose",
        "tests/unit"
      ],
      "console": "integratedTerminal",
      "internalConsoleOptions": "neverOpen",
      "envFile": "${workspaceFolder}/.env"
    }
  ]
}
```

### Como usar no VS Code

1.  Abra a pasta `SeniorFit` no VS Code.
2.  Vá para a aba "Executar e Depurar" (Ctrl+Shift+D ou Cmd+Shift+D).
3.  No menu suspenso de configurações de execução, você verá as opções:
    *   **Launch Backend**: Inicia o servidor Node.js e o depurador.
    *   **Launch Frontend**: Abre o frontend no Chrome (certifique-se de que o backend já esteja rodando).
    *   **Debug Unit Tests**: Executa os testes unitários com o depurador Node.js anexado.
4.  Para executar as tarefas (como iniciar o backend ou rodar todos os testes), vá para o menu "Terminal" > "Executar Tarefa..." e selecione a tarefa desejada.

## 8. Considerações Finais

Este projeto foi desenvolvido com a premissa de ser um MVP didático e testável. As escolhas de arquitetura e tecnologia visam a simplicidade e a facilidade de compreensão e manutenção, especialmente no contexto de uma avaliação acadêmica. A cobertura de testes unitários, de API e de UI garante que os requisitos funcionais e não funcionais (como as regras de acesso) sejam validados de forma abrangente.
